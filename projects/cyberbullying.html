PROJECT 03

AI Cyberbullying Detection Agent

A two-stage AI pipeline that detects cyberbullying in real time and generates culturally aware, age-calibrated interventions — moving beyond content moderation toward empathetic, preventative support.

Background

Most cyberbullying tools react after harm has already occurred — flagging or removing content without addressing the behavior behind it. For young people who spend significant time on digital platforms, this reactive approach leaves the root causes unresolved.

This project explored a different framing: instead of punishment, could an AI system intervene in the moment and redirect aggressors through empathy, education, and reflection? And to do that effectively, could it account for the cultural and demographic context of the person it's speaking to?

The research question driving the project: how can a culturally aware language model be designed to identify and intervene in cyberbullying scenarios in a way that is both preventative and empathetic?

Design

The system is organized into three layers that work together to understand the user, respond thoughtfully, and remain accountable.

CONTEXT-ENRICHMENT LAYER

Collects the user's message alongside basic demographic context — age and ethnicity — entered before the chat begins. This gives the model situational awareness: who is involved, what is being said, and the potential risk level. The cyberbullying classifier runs here first, identifying whether the message is harmful and, if so, what type (age-based, ethnicity-based, gender-based, religion-based, or other).

CULTURAL ATTUNEMENT & EMPATHY LAYER

Once a message is classified, this layer determines how to respond. Prompt engineering guides the model's tone, framing, and intervention type based on context and severity. Depending on the situation, the model can respond in four ways: reflective prompting for low-severity cases, resource redirection for context-specific offenses, empathy reframing for moderate-to-high severity, or a cooldown pause for repeated or escalating behavior.

HUMAN-IN-THE-LOOP LAYER

After each AI response, users rate the interaction on five dimensions: perceived empathy, cultural sensitivity, helpfulness, trust and safety, and personal experience. This feedback loop keeps the system grounded in real human judgment and provides structured data for continuous improvement rather than relying on automation alone.

Implementation

The prototype is a deployed web application with a two-stage chat interface. Development involved four stages: data preparation, model training, prompt engineering, and human validation.

  • Dataset: 47,000+ labeled Twitter posts from the Cyberbullying Classification dataset on Kaggle, covering six categories — age, ethnicity, gender, religion, other cyberbullying, and not cyberbullying. Preprocessed into JSONL format for fine-tuning.
  • Classifier: GPT-4o fine-tuned via OpenAI's API on the processed dataset to classify incoming messages by bullying type. Structured outputs enforced via OpenAI's JSON schema response format, keeping classification and generation as independently configurable, testable components.
  • Response generation: A GPT-4o-mini reasoning layer takes the classification output and user demographics and generates a tailored intervention. Prompts enforce cultural neutrality, age-appropriate tone, and non-judgmental framing, with curated support resources surfaced for context-specific offenses.
  • Frontend: Vue.js chat interface deployed on Firebase, with a pre-chat demographic input screen feeding context into the backend before the session begins.

Results

The evaluation used A/B testing — comparing the culturally tuned model against a base GPT model — measured across five user-rated dimensions.

  • Unexpected finding: The untuned base model outperformed the fine-tuned version across most dimensions, particularly cultural sensitivity (4.5 vs. 3.5) and perceived empathy (3.75 vs. 3.37). Users found the base model's responses more natural and relatable despite being more generic.
  • Trust and safety scored highest in both versions (4.0 and 4.16 respectively), suggesting the overall system design was perceived as safe and non-judgmental regardless of tuning.
  • Root cause analysis: The tuned model's underperformance likely stemmed from limited training data, high OpenAI API fine-tuning costs constraining dataset size to under 5% of available data, and prompting that was calibrated for younger teens but tested with university students.

The results reframed how the team thought about cultural tuning: it is not sufficient on its own without stronger human feedback loops, larger representative datasets, and more flexible prompting boundaries.

Closing

The project demonstrated that LLM-based cyberbullying intervention is technically feasible and perceived as trustworthy by users, but that cultural adaptation requires more than fine-tuning alone. The gap between a tuned and untuned model points toward future work on RLHF strategies, richer training data, and more nuanced prompt design across age groups.

More broadly, the system validates the idea that AI can play a genuinely supportive role in online safety — not just detecting harm, but actively fostering reflection and empathy at the moment it matters most.
